# Player Control 插件使用文档

## 插件简介

Player Control 插件允许拥有权限的玩家在 Minecraft 服务端上召唤和删除假人，并管理这些假人的权限和行为。假人会在被召唤后自动传送到召唤者的位置。

## 安装方法

1. 下载 `player_control.py` 文件并将其放入 MCDR 的 `plugins` 目录中。
2. 启动服务器或使用 `!!MCDR plugin load player_control.py` 命令加载插件。

## 配置文件

插件在首次加载时会自动生成两个配置文件：

1. **player_ops.json**：存储拥有召唤假人权限的玩家 ID 列表。
2. **dummy_list.json**：记录当前在线的假人及其召唤者信息。

这些文件存储在 `./config/player_control/` 目录下。

## 命令列表

### 1. 召唤假人

```
!!player spawn <假人ID> [<x> <y> <z>]
```

- **假人 ID**：必须参数，指定要召唤的假人的名称。
- **x, y, z**：可选参数，如果未提供，假人将会在默认坐标 (0, 1000, 0) 生成并自动传送到召唤者的位置。

### 2. 删除假人

```
!!player kill <假人ID>
```

- **假人 ID**：必须参数，指定要删除的假人的名称。

### 3. 假人丢弃物品

```
!!player drop <假人ID> <interval / 选项> <时间>
!!player dropstack <假人ID> <interval / 选项> <时间>
```

> drop子命令一次只会丢出一个物品，而dropstack子命令则会一次丢出整组物品。

- **假人 ID**：必须参数，指定要执行操作的假人的名称。
- **interval <时间>**：可选参数，设置丢弃物品的时间间隔。
- **mainhand** - 丢出主手上的物品。
- **offhand** - 丢出副手上的物品。
- **0~35** 的整数 - 丢出指定槽位的物品，从快捷栏第一格开始直到背包的最后一格。*注意，该指令疑似直接指定玩家背包的数组下标，因此物品栏序数* *从 0 开始* *，也就是 0 对应快捷栏第一格。*
- **all** - 丢出所有槽位的物品。注意，使用 `/player <target> drop all` 命令只会使玩家丢出背包中每种物品各一个。如需玩家丢出背包中所有物品，请使用 `/player <target> dropStack all` 命令。
- **continuous** - 使玩家不断丢出物品，类似按住丢弃键的操作。
- **interval** - 使玩家每隔一定游戏刻丢出一次物品。该参数要求后面再跟一个参数，指定丢出物品间隔的游戏刻数。如 `/player <target> drop interval 20` 命令会使得玩家每隔一秒丢出一次物品。
- **once** - 使玩家丢出一次物品，可用于重置 **continuous** 或者 **interval** 的丢弃间隔。

### 4. 假人切换物品栏

```
!!player hotbar <假人ID> <槽位>
```

- **假人 ID**：必须参数，指定要执行操作的假人的名称。
- **槽位**：必须参数，指定要切换到的物品栏槽位，可以是1~9的一个整数，对应着第一到第九个快捷栏。。

### 5. 假人交换主副手

```
!!player swapHands <假人ID>
```

- **假人 ID**：必须参数，指定要执行操作的假人的名称。

### 6. 假人转身

```
!!player turn <假人ID> back
!!player turn <假人ID> <俯仰角度> <右转角度>
```

- **假人 ID**：必须参数，指定要执行操作的假人的名称。
- **back**：让假人向后转身。
- **<俯仰角度> <右转角度>**：指定假人转身的具体角度。

### 7. 假人攻击

```
!!player attack <假人ID> <interval / 选项> <时间>
```

- **假人 ID**：必须参数，指定要执行操作的假人的名称。
- **continuous**：让玩家连续攻击或破坏，相当于按住左键。
- **interval**：让玩家每隔一定游戏刻攻击或破坏一次，相当于每隔一段时间按一下左键。该参数要求后面再跟一个参数，指定跳跃间隔的游戏刻数。
- **once**：与默认情况相同。

### 8. 假人使用物品

```
!!player use <假人ID> [interval <时间>] [<选项>]
```

- **假人 ID**：必须参数，指定要执行操作的假人的名称。
- **continuous**：让玩家连续使用，相当于按住右键。
- **interval**：让玩家每隔一定游戏刻使用一次，相当于每隔一段时间按一下右键。该参数要求后面再跟一个参数，指定跳跃间隔的游戏刻数。
- **once**：与默认情况相同。

### 9. 停止假人的行为

```
!!player stop <假人ID>
```

- **假人 ID**：必须参数，指定要停止行为的假人的名称。

### 10. 赋予玩家权限

```
!!player op <玩家ID>
```

- **玩家 ID**：必须参数，指定要赋予召唤假人权限的玩家名称。

### 11. 移除玩家权限

```
!!player deop <玩家ID>
```

- **玩家 ID**：必须参数，指定要移除召唤假人权限的玩家名称。

## 权限管理

- 管理员需要自己给自己权限。
- 插件通过 `player_ops.json` 文件来管理拥有召唤假人权限的玩家列表。
- 只有在 `player_ops.json` 中的玩家才可以使用召唤和管理假人的命令。
- 执行 `!!player op` 和 `!!player deop` 命令需要管理员权限（权限等级 4）。

## 日志监听

插件会监听服务端日志中的假人加入信息，识别到假人加入后会自动执行传送命令，将假人传送到召唤者的位置。

## 注意事项

- 所有命令和假人 ID 不区分大小写。
- 插件会自动处理日志中的大小写问题，确保假人名称匹配不区分大小写。
- 假人被召唤后会自动设置为生存模式。

## 卸载插件

要卸载插件，只需使用以下命令：

```
!!MCDR plugin unload player_control.py
```

插件卸载时会在服务器日志中显示 "Player Control 插件已卸载" 的消息。
